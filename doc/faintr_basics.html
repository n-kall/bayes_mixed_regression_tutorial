Comparing groups of factor levels with the faintr package

Motivation

The faintr package provides convenience function for the evaluation of a model fit, obtained with the brms package, for a Bayesian regression model for data from a factorial design. The faintr package allows extraction of many meaningful comparisons irrespective of the coding in the model. This is done by directly extracting the design matrix from the model and specifying the groups based on the data in, rather than the model coefficients.

Installation

Install the faintr package with devtools from GitHub:

devtools::install_github(
  "n-kall/faintr",
  build_vignettes = TRUE
)
library(faintr)

Example

Consider a data set on pitch frequency in the speech of female and male speakers in polite and informal contexts.

library(tidyverse)
data(politeness)

The cell means of this data set are:

politeness %>%
    group_by(gender, context) %>%
    summarize(mean_pitch = mean(pitch))
## # A tibble: 4 x 3
## # Groups:   gender [2]
##   gender context mean_pitch
##   <fct>  <fct>        <dbl>
## 1 F      inf           261.
## 2 F      pol           233.
## 3 M      inf           144.
## 4 M      pol           133.

A Bayesian regression model for a factorial design with by-subject and by-item random intercepts can be obtained with the brms package as follows:

library(brms)
m_dummy <- brm(pitch ~ gender * context + (1 | subject + sentence), politeness)

The brm function uses dummy coding per default. Look at the estimated coefficients:

fixef(m_dummy)
##                      Estimate Est.Error        Q2.5     Q97.5
## Intercept           261.46169 23.606153  212.429510 310.20295
## genderM            -116.81682 30.642542 -179.454820 -54.23636
## contextpol          -27.42428  7.846389  -43.213786 -11.79783
## genderM:contextpol   15.61385 11.307309   -6.199354  37.60235

The reference cell is where gender == F and context == inf, so female speakers in informal contexts. The estimated mean for the cell with data from male speakers in informal contexts is retrievable by adding the estimated coefficient genderM in the output above from the estimated Intercept.

The faintr package provides convenience functions to compare different (groups of) cells to each other, based on a model fit like the above. Although the fit of the regression model uses a particular reference cell for dummy-coding, other contrasts of relevance can be retrieved from the posterior samples. For example, if we want to compare two cell diagonally, say, male speakers in informal contexts against female speakers in polite contexts, we can do this by extracting the specified draws and then comparing them.

cell_definitions <- faintr::get_cell_definitions(m_dummy)

cell_draws <- faintr::get_cell_draws(m_dummy)

female_polite <- faintr::extract_draws(
  cell_definitions,
  cell_draws,
  gender == "F",
  context == "pol"
)

male_informal <- faintr::extract_draws(
  cell_definitions,
  cell_draws,
  gender == "M",
  context == "inf"
)

We can then calculate and visualise the difference between the specified groups.

library(ggplot2)

diff <- pull(female_polite) - pull(male_informal)

mean(diff)
## [1] 89.39255

qplot(diff, geom = "density")

Or we can plot the two posteriors for a visual comparison.


library(ggplot2)

bind_cols(female_polite, male_informal) %>%
  gather(key = "group") %>%
  ggplot(aes(x = value, color = group, fill = group)) +
  geom_density(alpha = 0.5)

Alternatively, we could compare males and females over both contexts. This is done by omitting the context variable from the specification.

female_overall <- faintr::extract_draws(
  cell_definitions,
  cell_draws,
  gender == "F"
)

male_overall <- faintr::extract_draws(
  cell_definitions,
  cell_draws,
  gender == "M"
)

bind_cols(female_overall, male_overall) %>%
  gather(key = "group") %>%
  ggplot(aes(x = value, color = group, fill = group)) +
  geom_density(alpha = 0.5)

This same procedure can be performed even if the model is created with treatment coding.

m_trt <- brm(pitch ~ 0 + gender * context + (1 | subject + sentence), politeness)
fixef(m_trt)
##                     Estimate Est.Error       Q2.5     Q97.5
## genderF            260.24740 23.015937 214.864262 306.10715
## genderM            144.66696 22.009134  99.689439 187.40214
## contextpol         -27.23565  8.045502 -42.862450 -11.97633
## genderM:contextpol  15.31718 11.497649  -5.995323  37.66045
cell_definitions <- faintr::get_cell_definitions(m_trt)

cell_draws <- faintr::get_cell_draws(m_trt)

female_polite <- faintr::extract_draws(
  cell_definitions,
  cell_draws,
  gender == "F",
  context == "pol"
)

male_informal <- faintr::extract_draws(
  cell_definitions,
  cell_draws,
  gender == "M",
  context == "inf"
)

bind_cols(female_polite, male_informal) %>%
  gather(key = "group") %>%
  ggplot(aes(x = value, color = group, fill = group)) +
  geom_density(alpha = 0.5)
